What we're typing

  • echo "# first" >> README.md
  • git init
  • git add README.md
  • git commit -m "first commit"
  • git remote add origin username.github.io
  • git push -u origin master

Setting your commit email as git email!

  • git config --global user.email "MY-NAME@example.com"

Adding in an actual webpage

  • Save a Sublime Text HTML file into you're folder and call it index.html

Pushing the changes

  • In your terminal make sure you're in the right folder: username.github.io
  • type: git status
  • type: git add .
  • type: git commit -m "adding index.html file"
  • type:git push

Day 6

Designing your personal websites

Hi humans!

Sorry I'm not there!

For today, each of you is going to design and possibly start coding your digital portfolio/personal website/coding journal! The point of this website is that it's going to be a place where you keep all your programming work. The point of it is to look awesome, give a sense of who you are and what you've made, and it has to work well on computers and phones (we'll go into this next time)

You should make at least 2 mock ups (i.e. hand-drawn blueprints like you did for your people cards) and hand them in! When you finish these, start coding a sample site or start working on mocking up a few specific ideas for longer-term individual projects! (e.g. an interactive story, a cool data representation, etc.)

Necessary supplies: Each of you needs blank paper and a ✍️ for mockups!

Things your website must include

  1. links to code "snippets and diddies" that you make.
  2. Some commentary and reflection about the things you're making!
  3. A cohesive aesthetic

Check out example websites...

For the following slides, go to the sites and "inspect the pages" to see the different parts of the sites and how they're made... (e.g.a menu bar is often just a div with links in it — on this site inspect the menu bar on the left in your console!)

Really look at the sites. Think about:

  • What is their structure (i.e. the divs/link/etc that they use.?
  • What is the information flow like?
  • Does moving through the site feel good to you? Does it match your aesthetic? Is it easy to navigate?
  • Does it showcase the projects well?
